# Git Grundlagen

## Kommandobeschreibungen

1. **git config** konfiguriert Einstellungen wie z. B. Benutzername und E-Mail-Adresse.
2. **git init** erstellt ein leeres Git-Repository.
3. **git commit** erstellt eine neue Version mit allen Änderungen.
4. **git status** gibt den aktuellen Status des Repositories samt geänderter Dateien aus.
5. **git add** fügt alle geänderten Dateien in die Staging Area hinzu.
6. **git log** zeigt alle älteren Versionen sowie alle commit-Nachrichten an.
7. **git diff** zeigt die Unterschiede zwischen der alten und der aktuellen Version an.
8. **git pull** lädt die Änderungen vom Remote-Repository herunter.
9. **git push** lädt die Commits auf das Remote-Repository hoch.
10. **git branch** erstellt einen Branch.
11. **git checkout** ändert den Branch, in dem wir uns befinden.
12. **git merge** fügt mehrere Branches zusammen.

## Auffälligkeit bei git status
Nach dem Hinzufügen der .gitignore Datei werden *.html sowie *.pdf-Dateien von Git nicht mehr angezeigt.
